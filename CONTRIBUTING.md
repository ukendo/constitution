# Як пропонувати зміни

Цей репозиторій містить **вимоги до статуту**. Зміни в `main` приймаються лише через pull request.

## Правила

1. **Одна вимога — один PR.** Правки до В-7 і В-16 — окремі PR (легше обговорювати й відкочувати).
2. Редакційні правки в кількох файлах (орфографія, посилання) можна об’єднати в один PR з міткою `редакція`.
3. Зовнішні учасники — через **fork** і PR. Write-доступ до репозиторію не потрібен.
4. Обговорення без готового тексту — у [Discussions](https://github.com/ukendo/constitution/discussions) або Issue з міткою `обговорення`.
5. Конкретну зміну формулювання — Issue / PR з міткою `вимога` (вкажіть номер: В-N).

## Порядок роботи

```text
1. Створіть гілку від актуального main
2. Змініть лише потрібні файли в vymohy/V-NN.md (і README, якщо змінюється короткий опис)
3. Відкрийте PR з короткою назвою: «В-7: …» або «редакція: …»
4. У описі PR поясніть *чому* змінюєте (не лише що)
5. Дочекайтеся рев’ю та merge
```

Після merge гілка видаляється автоматично.

## Мітки Issues / PR

| Мітка | Навіщо |
|-------|--------|
| `вимога` | Зміна конкретної вимоги (В-N) |
| `юридичне` | Потрібна юридична перевірка |
| `редакція` | Редакційні правки |
| `обговорення` | Питання без готового рішення |

## Версії

Зрізи фіксуються git-тегами (`v0.1`, `v0.2`, …) і згадуються в [CHANGELOG.md](CHANGELOG.md).
Повний знімок вимог для архіву: [`Вимоги_до_статуту_нової_федерації_v0.1.md`](Вимоги_до_статуту_нової_федерації_v0.1.md).

При розбіжностях між зібраним документом і файлами `vymohy/V-NN.md` пріоритет у **окремих файлів вимог**.
