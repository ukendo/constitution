# Як пропонувати зміни

Цей репозиторій містить **вимоги до статуту**. Зміни в `main` приймаються лише через pull request.

## Правила

1. У одному PR можна змінювати кілька вимог — як зручніше для роботи.
2. Зовнішні учасники — через **fork** і PR. Write-доступ до репозиторію не потрібен.
3. Обговорення без готового тексту — у [Discussions](https://github.com/ukendo/constitution/discussions) або Issue з міткою `обговорення`.
4. Для Issue / PR використовуйте мітки (`вимога`, `редакція`, `юридичне` тощо); у описі варто вказати номери вимог (В-N).

## Порядок роботи

```text
1. Створіть гілку від актуального main
2. Змініть потрібні файли в vymohy/<тема>/V-NN.md (і README, якщо змінюється короткий опис)
3. Відкрийте PR з короткою назвою (напр. «В-7, В-8: …» або «редакція: …»)
4. У описі PR поясніть *чому* змінюєте (не лише що)
5. Дочекайтеся рев’ю та merge
```

Після merge гілка видаляється автоматично.

## Мітки Issues / PR

| Мітка | Навіщо |
|-------|--------|
| `вимога` | Зміна вимог (В-N) |
| `юридичне` | Потрібна юридична перевірка |
| `редакція` | Редакційні правки |
| `обговорення` | Питання без готового рішення |

## Версії

Зрізи фіксуються git-тегами (`v0.1`, `v0.2`, …) і згадуються в [CHANGELOG.md](CHANGELOG.md).
Повний знімок вимог для архіву: [`Вимоги_до_статуту_нової_федерації_v0.1.md`](Вимоги_до_статуту_нової_федерації_v0.1.md).

При розбіжностях між зібраним документом і файлами `vymohy/<тема>/V-NN.md` пріоритет у **окремих файлів вимог**.
